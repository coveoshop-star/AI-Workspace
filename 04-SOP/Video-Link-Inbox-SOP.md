# Video Link Inbox SOP

## 目标

规范飞书 Knowledge Agent 收到 YouTube、视频链接或单独视频 ID 时的处理方式。

## 默认原则

如果机器人无法直接抓取标题、频道、简介或字幕，不要卡住流程。默认先把链接作为原始资料写入 `00-Inbox`。

## 默认动作

收到视频链接或视频 ID 时，先生成 Inbox 条目：

```text
00-Inbox/YYYY-MM-DD-video-<video-id-or-topic>.md
```

## 必填内容

Inbox 文件至少包含：

- 原始链接或视频 ID
- 来源渠道：飞书
- 发送时间
- 发送方标识，必要时脱敏
- 用户原始备注
- 当前抓取状态
- 需要人工确认

## 不要默认做的事

- 不要直接写入正式知识库。
- 不要把视频标题、频道、简介或字幕猜成事实。
- 不要在没有授权时安装 `yt-dlp` / `youtube-dl`。
- 不要在没有授权时启用浏览器自动化抓取。

## 需要用户确认后才能做

如果用户明确说：

- `可以抓标题`
- `可以抓字幕`
- `可以装 yt-dlp`
- `可以用浏览器`

才进入二次抓取流程。

## 二次抓取结果

抓取成功后，生成 Pending 审核稿：

```text
08-Pending/YYYY-MM-DD-video-<video-id-or-topic>-review.md
```

Pending 必须包含：

- 摘要
- 视频元数据
- 关键知识
- 可执行建议
- 建议写入位置
- 需要人工确认

## 飞书机器人推荐回复

```text
已收到视频资料。

状态：已先作为原始资料进入 00-Inbox
原因：当前无法直接确认标题/频道/字幕，避免把未知内容写成事实
下一步：
1. 你可以先补充一句这条视频大概讲什么
2. 如果需要我抓标题/简介/字幕，请回复“可以抓取视频信息”

需要人工确认：
- 是否只保留链接
- 是否抓取标题/简介/字幕
- 是否生成 Pending 综述
```

## 示例

视频 ID：

```text
K2a3oiD0fww
```

Inbox 建议文件名：

```text
00-Inbox/2026-07-01-video-K2a3oiD0fww.md
```
