# Daily Pending Review

每天晚上用这个清单审核 `08-Pending`。

## 审核时间

- 建议时间：每天晚上
- 建议时长：5-10 分钟
- 目标：只让高价值、可追溯、可执行的内容进入正式知识库

## 审核步骤

1. 打开 `08-Pending`。
2. 按文件更新时间查看当天新增内容。
3. 对每条 Pending 判断：approve / revise / archive。
4. approve 的内容写入正式目录。
5. revise 的内容保留在 `08-Pending` 并补充修改要求。
6. archive 的内容复制到 `13-Archive`。

## 审核标准

每条 Pending 必须回答：

- 来源是否明确？
- 是否与当前项目有关？
- 是否有长期保存价值？
- 是否有可执行建议？
- 是否需要人工确认？
- 应写入哪个正式位置？

## ChewyWell SEO 审核去向

ChewyWell SEO 相关内容优先写入：

## 审核结果标记

```md
# 审核结果

- [x] approve
- [ ] revise
- [ ] archive

# 审核备注

- 审核人：
- 审核时间：
- 入库位置：
- 需要后续动作：
```

## 不允许

- 不把无来源内容写入长期知识。
- 不把未确认事实写成结论。
- 不直接删除原始资料。
- 不让多个 Agent 同时写同一个正式文件。
